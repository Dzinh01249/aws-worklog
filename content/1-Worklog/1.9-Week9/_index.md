Week 9 Objectives:
- For the application to be capable of handling large-scale traffic and ensuring High Availability (HA), this week's goal is to establish an Application Load Balancer (ALB). The ALB will sit at the forefront as a gateway, receiving and evenly distributing network traffic from users to multiple backend EC2 servers behind it.

| Date | Tasks | Start Date | End Date | References |
|------|-------|------------|----------|------------|
| 12/06 | Learned the theory behind AWS Elastic Load Balancing. Proceeded to create a new Target Group, which will contain the destination servers (backend servers). Declared the protocol as HTTP on port 8080. Next, registered the existing EC2 server into this Target Group. Simultaneously, restructured the system by moving the EC2 server from the Public Subnet deep into the Private Subnet to enhance security, disallowing direct internet access. | 12/06 | 13/06 | |

If there is an error (crash, hang), the ALB will instantly route traffic away from that faulty server, avoiding impact on the end-user. | 16/06 | 17/06 | |

Week 9 Achievements:

- Successfully designed and deployed an industry-standard Load Balancing model.
- The system is no longer dependent on a Single Point of Failure, enhancing Fault Tolerance.
- Mastered the automatic Health Check mechanism, optimizing system maintenance procedures without causing service interruptions.
